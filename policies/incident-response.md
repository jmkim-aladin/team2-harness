# 장애 대응 프로세스

## 장애 등급

| 등급 | 기준 | 대응 시간 |
|------|------|-----------|
| P1 (Critical) | 서비스 전면 장애, 데이터 유실 위험 | 즉시 |
| P2 (Major) | 핵심 기능 장애, 일부 사용자 영향 | 1시간 이내 |
| P3 (Minor) | 비핵심 기능 장애, 우회 가능 | 업무 시간 내 |

## 대응 흐름

```
장애 인지
↓
영향 범위 파악 (서비스 하네스의 의존 시스템 확인)
↓
즉시 대응 (롤백 / 핫픽스 / 트래픽 전환)
↓
YouTrack 장애 티켓 생성
  - 재현 조건
  - 영향 범위 (사용자 수, 데이터 영향)
  - 원인 분석
  - 조치 내역
↓
근본 원인 분석 (5 Whys)
↓
재발 방지책 → 서비스 하네스 갱신
  - AGENTS.md 주의사항 추가
  - RUNBOOK.md 대응 절차 보강
  - smoke test 항목 추가
```

## 사후 리뷰

- P1/P2 장애는 반드시 사후 리뷰 실시
- 리뷰 결과를 서비스 하네스에 반영
- 같은 유형의 장애가 반복되지 않도록 가드레일 추가
