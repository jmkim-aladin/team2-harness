# 릴리즈/배포 정책

## 배포 원칙

- 프로덕션 배포는 반드시 사람 승인을 거친다
- 배포 전 smoke test 통과 필수
- 배포 후 즉시 검증 포인트 확인 (로그, 결과 데이터, 후속 배치)
- 롤백 절차를 PR에 명시한 상태에서만 배포 가능

## 배포 프로세스

```
코드 머지 → staging 자동 배포 → staging 검증
→ 프로덕션 배포 요청 → 승인자 확인 → 프로덕션 배포
→ 배포 후 smoke test → 이상 시 즉시 롤백
```

## 배포 시간대

- 일반 배포: 업무 시간 내 (팀 합의에 따름)
- 위험 배포 (DB/SP 변경 포함): 트래픽 적은 시간대 권장
- 핫픽스: 시간 제한 없음, 단 사후 리뷰 필수

## 롤백 기준

- 배포 후 5분 이내 주요 에러율 급증 시 즉시 롤백
- 롤백 방법은 PR 체크리스트에 사전 명시
- DB/SP 변경이 포함된 경우 롤백 스크립트 필수 첨부
