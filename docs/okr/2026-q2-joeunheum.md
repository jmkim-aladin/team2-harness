# 조은흠 2026 2분기 개인 OKR (초안)

> 상위: REF-A-3122 (2분기 팀 OKR) | REF-A-2175 (연간 팀 OKR)

## Objective

바자르 통합 관리자 서비스와 B2B 스토어프론트의 프론트엔드를 구축하고, 디자인 시스템·SDUI 기반 개발 프로세스를 안착시켜 팀의 프론트엔드 품질과 개발 속도를 동시에 끌어올린다.

## Key Results

### KR1. 바자르 통합 관리자 서비스 프론트엔드 구축 (4월)

> 팀 KR1 연계 — 주문/CS 처리를 위한 통합 관리자 서비스 구축 및 고우선순위 기능 10개 제공

- 통합 관리자 FE 프로젝트 초기 세팅 완료 (모노레포·Storybook·Vitest 구조 적용)
- 고우선순위 기능 중 FE 담당 화면 **4개 이상** 개발 완료 (4월 내)
- 개발 화면 단위 테스트 커버리지 **50% 이상** 확보
- 디자인 시안 대비 구현 일치율 **100%** 유지

### KR2. 디자인 시스템 컴포넌트 확장 및 Storybook 등록 (4월~)

> 팀 KR4 연계 — 디자인 시스템 필수 컴포넌트 15종(누적) 정의 및 Storybook 등록
> 팀 KR6 연계 — B2B 스토어프론트 디자인시스템 구축

- 1분기 10종 + 신규 **5종** 추가 → 누적 **15종** Storybook 등록 완료 (4월 4주차)
- B2B 스토어프론트 공통 컴포넌트 **5종 이상** 식별 및 설계 문서 작성
- 컴포넌트별 Storybook 스토리 **3개 이상** (default, variant, edge case)
- 디자이너 검수 완료율 **100%** (등록 컴포넌트 전수 검수)

### KR3. B2B 스토어프론트 FE 구조설계 (4~5월)

> 팀 KR6 연계 — KJM의 데이터 스키마·권한 격리 아키텍처와 동시 진행

- 멀티 테넌시 FE 라우팅·권한 구조 설계 문서 **1건** 완료 (4월, KJM 스키마와 병행)
- B2B 화면 컴포넌트 구조 정의: 페이지 템플릿 **5종 이상** 설계 (5월 2주차)
- 외주 개발자용 FE 보일러플레이트 **1세트** + 개발 가이드 문서 **1건** 완성 (5월 4주차)
- 외주 투입용 5W1H 기반 FE 티켓 **15개 이상** 사전 발행
- KJM BE 설계와 동기화 미팅 **격주 1회** 이상 수행

### KR4. SDUI 기반 이벤트 관리 시스템 고도화 (6월)

> 팀 KR4 연계 — SDUI 기반 이벤트 관리 시스템을 Admin으로 제공, 개발 요청 0% 자립화

- SDUI 이벤트 관리 시스템 Admin 통합: 운영자 사용 가능한 화면 **5개 이상** 제공
- 만권당·투비 서비스 **2개** 모두 적용 완료
- 운영자 자립 테스트: 단순 이벤트 페이지 생성 성공률 **100%** (최소 3건 검증)
- 개발팀 이벤트 페이지 구성 요청 **0건** 달성 (6월 4주차 기준)
- 운영자 대상 사용 가이드 문서 **1건** 작성
