# 개발2팀 2026 1분기 OKR

> YouTrack KB: REF-A-2470 | 작성자: 김규태

## Objective

바자르·만권당·AI·보안 거버넌스 강화라는 연간 목표 달성을 위해, 1분기 내 기술적 기반과 운영 체계를 정비하여 성장 가속을 위한 준비 단계를 완성한다.

## Key Results

### KR 1. 바자르 매출확대를 위한 구조적 기반 확립

- 상품/주문/배송 도메인 운영 예외로그 월 5건 이하 유지
- 오픈마켓 3곳(G마켓, 옥션, 11번가) 연동 완료
- 상품 자동 등록 성공률 95% 이상, 10만권 등록 완료

### KR 2. 만권당 코드베이스 전환을 위한 마이그레이션 준비 완료

- 신규 주문 시스템 모듈 개발 100% 완료
- 운영 반영 후 4주간 가용율 99.5% 이상 유지하고, 치명적 장애 0건

### KR 3. AI 기반 생산성 강화를 위한 사전 체계 구축

- 핵심 기능 문서화 30% 달성
- 테스트 커버리지 개선 우선순위 도출
- AI 기반 자동화 목표 도출 및 PoC 1회 진행, 보고서 1건 작성

### KR 4. 고객 경험 및 디자인 일관성 강화

- 만권당 디자인 시스템 1차 컴포넌트 10종 정의
- 화면 개발 프로세스 표준화 문서 완성

### KR 5. 보안·거버넌스 체계 강화

- 나루 서비스 활용을 위한 DPA 템플릿 작성 및 정보보안팀 검토 완료
- 팀 운영 서비스 전체 대상 SonarQube 혹은 Fortify 취약점 검사 월 1회 보고서 작성, Critical/High 이슈 0건 유지

## 개인별 OKR

### 조윤주

**Objective**: 바자르 오픈마켓 확장에 대응하고 향후 AI 기반 생산성 강화를 준비하기 위해, UX·운영·테스트의 표준을 정립하고 적용한다.

**KR1. 바자르 오픈마켓 매출 확대를 위한 UX·운영 기준 정립 및 적용**
- 오픈마켓 상품 자동 등록을 위한 몰타입별 규칙 정의 및 테스트 완료 (G마켓·옥션·11번가)
- 상품 등록 실패 원인 유형화 및 모니터링 체계 정립, 분기 내 자동 등록 규칙 개선 사항 1건 이상 반영
- 오픈마켓별 상품 상세 정보 연동 필수/선택 항목 정의 및 상세 정보 스타일 확정·적용
- CS/교환·반품 처리 중심의 UX 시나리오 설계

**KR2. 주요 기능의 '표준 테스트 케이스' 정의 및 실제 운영 적용**
- B2B 제휴 개발 시 활용 가능한 표준 테스트 케이스 정의 (계정/상품/주문 핵심 시나리오 포함)
- 1분기 내 배포되는 신규·개선 기능 테스트 케이스 적용률 100%
- 만권당·투비컨티뉴드 핵심 기능 선정 및 테스트 케이스 작성 후 팀 내 배포

### 김정실

**Objective**: 이벤트 페이지 제작 과정에서 시맨틱·접근성 마크업 기준을 명확히 정립하고, 이를 실제 퍼블리싱에 적용하여 페이지 품질을 안정적으로 향상한다.

**KR1.** 1분기 내 이벤트·기획전 페이지에 적용 가능한 시맨틱 마크업 및 웹 접근성 기준을 정리한 가이드·매뉴얼·체크리스트를 제작하고 배포한다.

**KR2.** 1분기 동안 진행되는 이벤트 페이지 퍼블리싱 작업을 대상으로, 정의된 시맨틱 구조·접근성 체크리스트 충족률을 95% 이상 유지한다.

### 이유림

**Objective**: AI 활용 실험, 디자인 시스템 구축, 템플릿 표준화를 통해 확장 가능하고 지속 가능한 디자인 운영 체계를 목표로 한다.

**KR1.** AI 활용 실험 프로젝트 및 케이스 2건 이상 적용
**KR2.** 투비(ToBe) 디자인 시스템 구축 기초를 위한 투비 시스템 파악 (디자인 시스템 가이드 30%)
**KR3.** 만권당 이벤트 페이지 템플릿 제공 1건

### 이현민

**Objective**: 멀티캠퍼스 신규 고객사 런칭, 코틀린 기반 바자르 개발 구조 이해, AI 활용 생산성 및 안정성 향상

**KR1. 멀티캠퍼스 런칭 및 운영 안정화**
- 1분기에 멀티캠퍼스 고객사를 런칭하고 운영을 안정화하여 B2B업무에 활용 가능한 시간을 확보
- 멀티캠퍼스 연동 로직을 문서화하여 운영 처리 시간을 단축

**KR2. 바자르 운영·연동 업무 수행 및 문서화**
- 바자르 주요 모듈을 분석 및 파악하고 AI를 활용하여 구조 파악 생산성 향상
- 주요 모듈 관계에 대해 플로우/주요 클래스 관계를 다이어그램 또는 문서로 정리
- 바자르 관련 운영 이슈(버그/오류 티켓)를 직접 분석·해결하고, 원인·해결 방안 문서 정리

### 안혜련

**Objective**: 바자르 서비스의 차세대 개발(Kotlin) 표준을 수립하고, 운영 효율화를 통해 개발 리소스를 최대화한다.

**KR1. 바자르 현행 시스템 분석을 통한 '코틀린(Kotlin) 개발 표준 가이드 (Ver 0.5)' 수립**
- 기존 바자르 소스 코드 아키텍처 및 패턴 분석
- 분석 기반의 코딩 컨벤션 및 개발 가이드라인 정의 (Ver 0.5)

**KR2. 바자르 개발 리소스 확보를 위한 운영 효율화**
- 전사 공용 성인인증 API 개발로 바자르 등 신규 서비스 적용 시 개발 공수 Zero화
- 서재 서비스 인프라(AWS) 이관 완료로 운영 불확실성 제거
- 납품 프로세스 병목 개선 및 오류 제로화 달성

### 김정민

**Objective**: 만권당 정량제 서비스를 출시하고 ADMIN MSA를 분리하며, 코드 품질 및 보안 관리 체계의 기반을 마련한다.

**KR1. 만권당 정량제 서비스 출시**
- 정량제 서비스 운영 환경 배포 완료 및 안정성 확보
- 열람 제한 설정 / 제한 기능 정상 동작 (QA 통과)
- B2C/B2B 구분 처리 완료, 정산 연동 완료, 서비스 장애 0건 유지

**KR2. 만권당 ADMIN MSA 서비스 오픈**
- ADMIN Backend/Frontend 운영 환경 배포 완료
- 멀티모듈 구조 구축, 사업부 요구 핵심 기능 3개 이상 구현, 서비스 장애 0건 유지
- ADMIN 개발 가이드 문서 작성

**KR3. 만권당 정산 MSA 아키텍처 설계 완료**
- 정산 설계 문서 4종 작성 완료 (레거시 정산 분석, 정산 MSA 아키텍처, 도메인 모델 설계, 인프라 설계)

**KR4. 코드 품질 관리 기반 구축**
- 만권당·투비 P0 SP 100% Git 등록 완료
- PR 기반 SP 변경 비율 80% 이상 달성
- AI 코드 리뷰 자동화 파일럿

**KR5. 보안 기반 구축**
- DPA 기술 요건 및 계약 초안 작성 완료

### 조은흠

**Objective**: 만권당 어드민 Frontend 개발을 완수하고, 개발 과정에서 Storybook과 테스트 환경을 구축하여 프론트엔드 품질 관리 체계의 기반을 마련한다.

**KR1. 만권당 어드민 Frontend 개발 및 배포 완료**
- 모노레포 내 만권당 어드민 프로젝트 생성 및 운영 환경 배포 완료
- 관리자 핵심 화면 구현 완료

**KR2. Storybook 기반 컴포넌트 문서화**
- 만권당 어드민 프로젝트에 Storybook 환경 구축
- 만권당 어드민 컴포넌트 10개 이상 Storybook 문서화 완료
- 디자이너 검수용 Storybook 공유 프로세스 수립

**KR3. 테스트 환경 구축 및 적용**
- Vitest 단위 테스트 환경 적용
- 만권당 어드민 주요 기능 단위 테스트 커버리지 50% 이상 달성
- 테스트 작성 가이드 문서 작성
