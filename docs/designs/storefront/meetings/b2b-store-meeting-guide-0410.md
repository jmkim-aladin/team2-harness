# B2B 전용몰 Scope 설정 회의 진행 가이드

> 회의일: 2026-04-10(목) 15:00
> 목적: 프로젝트 방향성 및 범위(Scope) 설정
> 참석자: 김규태(팀장), 조윤주(기획), 김정민(아키텍처), 조은흠(FE), 안혜련(B2B BE), 이현민(B2B BE)

## 현재 위치

```
[완료] 정의·목적           → [오늘] 방향성·범위          → [다음] 기술 설계
      기획자 정리                 Scope 설정                   이벤트 스토밍
      As-Is / To-Be              뭘 만들고 뭘 안 만들지         어떻게 만들지
```

## 회의 원칙

- **"뭘(What)" 먼저, "어떻게(How)"는 다음에**
- 기술 이야기(DB, API, 아키텍처)로 빠지지 않기
- 의견이 갈리면 적어두고 다음 항목으로
- 모르는 건 "모른다"로 적고 담당자 배정

## 역할 분배

| 참석자 | 역할 |
|--------|------|
| 김규태 (팀장) | 의사결정, MVP/나중에 최종 판단 |
| 조윤주 (기획) | 회의 리드, 요구사항 정리, 산출물 작성 |
| 안혜련, 이현민 (B2B BE) | 현행 B2B 시스템/운영 실태 공유 (기존 레거시 담당자) |
| 조은흠 (FE) | FE/화면 관점 의견 |
| 김정민 (아키텍처) | 가이드 + "가능/어려움" 한 줄 판단 |

## 진행 순서 (총 90분)

### Step 1. 확정 사항 확인 (5분)

지난 회의에서 합의된 내용을 화면에 띄우고 빠르게 확인.

> **프로젝트 정의**: B2B 고유의 거래 구조와 정책을 독립적으로 수용하기 위한 전용 커머스 플랫폼
> **핵심 컨셉**: 운영자가 개발자 도움 없이 설정만으로 제휴사별 맞춤형 몰을 즉시 생성

"이건 지난번에 합의한 것입니다. 오늘은 여기서부터, **구체적으로 뭘 만들 건지** 범위를 정하겠습니다."

---

### Step 2. 기존 B2B 현실 공유 (15분)

**안혜련/이현민(기존 B2B 레거시 개발자)이 주도.** 현재 시스템과 운영 실태를 공유합니다. 조윤주(기획)가 비즈니스 관점을 보충합니다.

질문 리스트 (화이트보드에 적어두면 좋음):

| # | 질문 | 누가 답하나 | 답변 |
|---|------|-----------|------|
| 1 | 현재 B2B 제휴사는 몇 곳? | 안혜련/이현민 | |
| 2 | 제휴사마다 가장 많이 다른 점 3가지는? | 안혜련/이현민 | |
| 3 | 제휴사 하나 추가할 때 코드 수정이 어디에 필요? 얼마나 걸리나? | 안혜련/이현민 | |
| 4 | 개발자가 가장 불편한 점은? (커스텀 개발 관점) | 안혜련/이현민 | |
| 5 | 기획/운영 관점에서 가장 불편한 점은? | 조윤주 | |
| 6 | 제휴사별 설정이 코드에 박혀 있나, DB에 있나? | 안혜련/이현민 | |
| 7 | 설정을 바꾸려면 배포가 필요한가? | 안혜련/이현민 | |

> 이 답변이 "어떤 설정을 먼저 만들지"의 근거가 됩니다. 안혜련/이현민이 레거시를 가장 잘 아는 사람들입니다.

---

### Step 3. "운영자가 설정할 수 있어야 하는 것" 나열 (20분)

프로젝트 정의가 "설정만으로 맞춤형 몰 생성"이니까, **구체적으로 뭘 설정할 수 있어야 하는지** 나열합니다.

**진행 방법:**

1. 각자 포스트잇에 "운영자가 설정할 수 있어야 하는 것" 작성 (3분, 조용히)
2. 전체 공유하면서 화이트보드에 붙임
3. 중복 합치고, 빠진 것 추가

**예상 항목 (참고용, 회의에서 나올 것):**

- 제휴사별 노출 상품 범위 (카테고리 선택)
- 제휴사별 가격/할인율
- 결제 수단 제한 (포인트, 카드, 후불 등)
- 포인트/마일리지 적립 정책
- 실무 결제 제한 (금액 한도, 부서별 예산)
- 주문 승인 프로세스 (금액별 승인)
- 화면 구성/랜딩페이지
- 노출 제한 (특정 카테고리 제외)
- 배송지/배송 조건
- 주문 단위 (최소 수량 등)

> **이 단계에서 "기술적으로 어렵다/쉽다"는 아직 말하지 않습니다.** 전부 꺼낸 다음에 구분합니다.

---

### Step 4. MVP vs 나중에 (20분)

나열된 항목을 3개 그룹으로 나눕니다.

```
┌──────────────────┬──────────────────┬──────────────────┐
│  MVP              │  Phase 2          │  미래             │
│  없으면 런칭 불가   │  있으면 좋지만      │  기존에도 없던 것   │
│                   │  수동 해결 가능     │                   │
├──────────────────┼──────────────────┼──────────────────┤
│                   │                   │                   │
│                   │                   │                   │
│                   │                   │                   │
│                   │                   │                   │
└──────────────────┴──────────────────┴──────────────────┘
```

**판단 기준:**
- "이게 없으면 첫 번째 신규 제휴사를 온보딩할 수 없나?" → **MVP**
- "이게 없어도 수동으로 해결 가능하나?" → **Phase 2**
- "기존에도 없던 건데?" → **미래**

**김정민은 이때 한 마디씩만:**
- "이건 간단함" (테이블 + CRUD 수준)
- "이건 좀 걸림" (별도 로직/연동 필요)
- "이건 별도 설계 필요" (모듈/엔진 설계가 먼저)

깊은 기술 논의는 하지 않습니다.

---

### Step 5. 다음 스텝 합의 (10분)

| 항목 | 내용 |
|------|------|
| MVP 항목 합의 | 확정 / 추가 확인 필요 |
| 확인 필요 사항 | 담당자 + 기한 배정 |
| 다음 회의 | 일시, 아젠다 (MVP 기반 기술 설계) |

---

## 회의 중 주의사항

| 빠지기 쉬운 함정 | 대응 |
|-----------------|------|
| "DB는 어떻게..." | "그건 범위가 정해진 다음에. 오늘은 뭘 만들지만." |
| "바자르 API가..." | "기술 제약은 다음에. 오늘은 비즈니스 기준으로." |
| 한 항목에서 30분 토론 | "의견이 갈리면 적어두고 다음 항목으로." |
| "이것도 만들면 좋겠다" | "좋은 아이디어. 'Phase 2' 칸에 적어두자." |

---

## 회의 산출물 (이것이 나와야 성공)

```
[MVP 범위]
- 항목 A
- 항목 B
- ...

[Phase 2]
- 항목 C
- 항목 D
- ...

[확인 필요]
- 항목 E: 확인 내용 (담당: OOO, 기한: O/O)
- ...

[다음 회의]
- 일시:
- 아젠다: MVP 항목 기반 기술 설계
```

## 사전 준비 체크리스트

### 안혜련/이현민 (회의 전 준비)

- [ ] Step 2의 질문 7개에 대한 답변 준비
- [ ] 제휴사별로 커스텀 개발했던 구체적 사례 1~2개 준비
- [ ] 현재 B2B 코드에서 제휴사별 분기가 있는 부분 파악

### 조윤주 (회의 전 준비)

- [ ] 기획/운영 관점에서 제휴사 관련 불편 사항 정리
- [ ] "운영자가 설정할 수 있어야 하는 것" 본인 의견 준비

### 김정민 (회의 전 준비)

- [ ] 바자르 상품/주문 REST API 존재 여부 확인 (배치 외)
- [ ] Naru partner 테이블이 B2B 테넌트로 활용 가능한지 확인
- [ ] 뉴빌링 B2B 결제 수단 지원 범위 확인

> 김정민의 사전 조사 결과는 회의에서 직접 발표하지 않습니다. Step 4에서 "가능/어려움" 판단할 때 내부적으로 참고합니다. 기술 논의가 필요하면 다음 회의에서.

### 조은흠 (회의 전 준비)

- [ ] 현행 B2B 화면이 있다면 확인
- [ ] "제휴사별 화면 구성"이 나왔을 때 의견 준비 (템플릿? SDUI? 설정?)
