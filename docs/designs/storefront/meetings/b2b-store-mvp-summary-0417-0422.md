## 1. 첫 고객

**테넌트 고객 (도서 구매자)** — 제휴사 고객 또는 대량구매 고객
**확정 시점**: 차주 내(18주차)

**성공 정의**: "테넌트 고객이 도서 N권을 주문해서 받고, 필요 시 취소할 수 있다."

---

## 2. MVP 포함 기능

### 전시

| 도메인 | MVP 범위 | 비고 |
|---|---|---|
| 카탈로그(상품) | 바자르 연동 + 가격 표시 | |
| 가격 | 정가 + 테넌트별 고정 할인율 | 판매가, 공급가 기준 설정 가능 |
| 재고 | 바자르 실시간 조회 | 바자르 자체 재고 로직 적용 |
| 전시 | 기본 랜딩 + 카테고리 + 큐레이션 + 베스트셀러 | 최소 전시 |
| 검색 | 알리스 검색엔진 단순 검색 | 카탈로그 기반 필터, 전시 연계 필수 |

### 주문

| 도메인 | MVP 범위 | 비고 |
|---|---|---|
| 장바구니 | 세션 기반 장바구니 | |
| 주문 | 주문서 생성 + 정책 검증 | 바자르 배송 그룹 정책상 주문 분리 필요 고려 |
| 결제 | 뉴빌링 단건결제 + 제휴사 포인트 단건 = 복합결제 | 첫 고객사별 결정 |
| 배송 | 상태 추적 + 단일 배송지 | 멀티배송지는 구조만 준비, 첫 고객사별 결정 |
| 클레임 | 취소·부분취소만 | 반품·교환은 수동 대응 → Phase 2 |

### 구매 후

| 도메인 | MVP 범위 | 비고 |
|---|---|---|
| 혜택 | 제휴사 전용 포인트(단건) | 알라딘 포인트 미적용 |
| 알림 | 알라딘 기존 알림 연동 (주문확인·배송) | 알라딘 기존 알림 연동 외 법적 리스크 검토 후 확정 |

### 인프라

| 도메인 | MVP 범위 | 비고 |
|---|---|---|
| 로그인·권한 | 나루 로그인 | 제휴사 연동 프로토콜 고려 |
| 테넌트 관리 | 1개 테넌트 기본 구조 | |
| 모니터링 | 주문현황·주문량 뷰 | 제휴몰 관리자 열람 |

## 3. MVP 관련 기능 범위

MVP에 포함된 기능을 **각 담당자가 설정·조회하는 부분까지 개발 포함**.
