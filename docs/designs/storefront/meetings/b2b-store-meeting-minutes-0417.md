## 1. 확정 사항

### 1.1 도메인 분류 체계 — 13개 영역 공식화

구매 여정 기반으로 **"전시 / 주문 / 구매 후"** 3개 영역 × 13개 도메인 확정.

| 영역 | 도메인 | 설명 |
|------|--------|------|
| 전시 | 카탈로그(상품) | 판매 가능한 상품의 원천 데이터 관리 (상품명·저자·출판사·카테고리·이미지) |
| 전시 | 가격 | 정가·할인율·판매가 계산 |
| 전시 | 재고 | 판매 가능 수량 관리, 품절 처리, 수량 제한 정책 |
| 전시 | 전시 | 카테고리 구성·노출 순서·기획전·배너 등 편성 |
| 전시 | 검색 | 키워드·필터 기반 상품 탐색 |
| 주문 | 장바구니 | 구매 전 임시 보관, 수량·삭제·가격 미리보기 |
| 주문 | 주문 | 구매 확정 및 주문서 생성 (체크아웃 흐름 전반) |
| 주문 | 결제 | PG 연동, 결제수단별 처리, 성공·실패 핸들링 |
| 주문 | 배송 | 배송지·배송비·배송 방식·상태 추적 |
| 주문 | 클레임 | **취소·반품·교환 통합**, 부분 취소, 환불 정책 |
| 구매 후 | 정산 | 주문 기반 수익 집계·수수료 차감·지급액 산출 |
| 구매 후 | 혜택 | 적립금·마일리지·쿠폰 등 보상 체계 |
| 구매 후 | 알림 | 이벤트 기반 커뮤니케이션 (주문 확인·배송·클레임) |

> 영향: `decisions.md` 및 `event-storming-*` 문서의 **13영역 + 플랫폼 레이어 3** 구조 공식 확정. 기존 D1~D8 정의는 이 구조로 대체됨.

## 3. B2C 대비 B2B 특화 정리

### 3.1 B2B에서 제외할 B2C 기능

| 기능 | 판단 | 근거 |
|------|------|------|
| 추가 주문 (직후 묶음 배송) | **우선순위 제외** | B2B 환경에서 쓰이지 않거나 불필요 |
| 분리 배송 | **제공 안 함** | 발송일 다른 상품 따로 받는 옵션은 B2B 제휴 고객에게 차단하는 경우 많음 |

### 3.2 계약 의존 — 회원 등급·마일리지

- B2C 계정 연동 시 **마일리지 적립·회원 등급 산정 대상 포함 여부**가 계약에 따라 달라짐
- → `decisions.md` D-12(결제수단·세금계산서) + D-13(계약 관리)와 연동 결정 필요

### 3.3 B2B 특화 UX — "무결제" 프로세스

- 요청: **포인트 전액 결제 시 다른 결제 수단이 노출되지 않도록** 하는 UX
- → `decisions.md` D-12에 결제수단 노출 정책으로 편입
