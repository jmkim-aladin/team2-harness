## 1. 역할 정의

MVP에서 등장하는 설정·조회 주체는 3명.

| 역할 | 설명 | 접근 채널 |
|---|---|---|
| **플랫폼 운영자 (내부)** | 알라딘 개발2팀/운영팀. 테넌트 생성, 계약 반영, 가격 정책, 포인트 어댑터 설정 등 플랫폼 공통 설정 담당 | 내부 어드민 |
| **제휴몰 관리자 (테넌트 측)** | 제휴사(또는 대량구매 고객사)의 몰 운영 담당자. 자사 회원 정책·주문 조회·기본 정보 열람 | 제휴몰 어드민 |
| **구매자 (임직원/대량구매)** | 도서를 실제로 구매하는 최종 사용자 | 스토어프론트 |

> 구매자 플로우는 DEV2-A-1087 §2에 이미 정의됨. 이 문서는 **플랫폼 운영자**와 **제휴몰 관리자** 두 주체를 다룬다.

## 3. MVP 범위 선긋기 — 무엇을 "담당자 설정"에 넣을 것인가

### ✅ MVP 포함 (UI까지 구현)

**플랫폼 운영자 어드민**
- 테넌트 생성·수정 (기본 정보·계약·할인율·포인트 계약)
- 서비스 구독 활성화 (도서몰 하나)
- 제휴사 연동 프로토콜·포인트 어댑터 설정
- 카테고리별 정책 기본값·상품 노출 범위
- 전시 기본 구성 (랜딩·카테고리·큐레이션·베스트셀러 규칙)
- 결제·배송·알림 기본 정책
- 정산 CSV export

**제휴몰 관리자 어드민**
- 회원별 구매·배송 제한 정책 CRUD
- 자사 회원 목록 조회
- **주문현황·주문량 뷰** (MVP 핵심 신규)
- 자사 주문·취소 내역 조회
- 계약·할인율·결제수단·정책 **읽기 전용 조회**
- 알림 종류 on/off

### 🟡 MVP 경량 (최소 UI or 수동)

- 알림 법적 리스크 검토 후 Opt-out 설정 필요 여부 재확인
- 정산은 CSV export만, 자동 집계·대시보드 X
- 복합결제 "뉴빌링 + 제휴 포인트" 구성은 하드코딩에 가까운 형태 허용 (첫 고객사 확정 후 UI 정교화 재검토)

### ❌ MVP 제외 (Phase 2+)

- SDUI 기반 UI 커스텀
- 수기 조직 인증 프로세스
- 허용 불가 품목 감지·이상탐지
- 제휴사 전용 URL·SSO 관리 UI
- 계약 관리 Write UI (MVP는 하드코딩 기반 Read)
- CS 내장 도구
- 쿠폰·포인트 원장

## 4. 설계 파급

### 신규·변경 BC / 기능

| 대상 | 변경 내용 |
|---|---|
| Tenant Management | 플랫폼 운영자 어드민 UI (테넌트 CRUD·계약·할인율·포인트 계약·서비스 구독) |
| Member/Organization | **제휴몰 관리자 UI (회원 정책 CRUD·회원 목록)** — D-21 회원별 제한 정책 반영 |
| Display | 플랫폼 운영자의 랜딩·카테고리·큐레이션 구성 어드민 (최소) |
| Payment Orchestration | 포인트 어댑터 설정 어드민 (제휴사 전용 포인트) |
| Monitoring | **제휴몰 관리자용 주문현황·주문량 뷰 (MVP 신규 핵심)** |
| Settlement | 플랫폼 운영자 CSV export |

### 권한 모델 영향

- 플랫폼 운영자 / 제휴몰 관리자 **2-tier 권한 구분 필수** (MVP)
- 제휴몰 관리자는 **자사 데이터만** 접근 (테넌트 격리)
- 구매자 본인 주문·알림은 스토어프론트 권한

### 어드민 UI 구조 제안

```
내부 어드민 (플랫폼 운영자)
├─ 테넌트
│  ├─ 목록·생성
│  ├─ 상세 (기본 정보·계약·할인율·포인트 계약·서비스 구독)
│  └─ 연동 설정 (인증·포인트 어댑터)
├─ 카탈로그·가격 정책
├─ 전시 구성
├─ 결제·배송·알림 기본 설정
└─ 정산 CSV export

제휴몰 어드민 (제휴몰 관리자, 테넌트 격리)
├─ 회원
│  ├─ 목록
│  └─ 회원별 구매·배송 제한 정책
├─ 주문현황·주문량 (읽기 전용, Monitoring BC)
├─ 주문·취소 내역 (자사 범위)
├─ 알림 on/off
└─ 기본 정보 조회 (계약·할인율·결제수단·정책)
```

## 5. 미결 사항

- [ ] **제휴몰 어드민 접근 경로** — 스토어프론트와 통합 도메인? 별도 하위 경로? 제휴사별 URL?
- [ ] **제휴몰 관리자 계정 인증** — 나루 통합? 별도? (인증 구조와 D-11 RBAC 연동)
- [ ] **주문현황·주문량 뷰 집계 주기** — 실시간 vs 배치? 개인정보 노출 범위?
- [ ] **대량구매 고객사**가 첫 고객이 될 경우, 제휴몰 관리자 역할이 달라질 가능성 (18주차 고객 확정 후 재확인)
- [ ] 알림 on/off 단위 — 테넌트 전체? 회원별? (법적 리스크 검토와 연동)
- [ ] **정산 CSV 포맷 스펙** — 누가 받는가(운영팀? 제휴사 재무?)·주기·컬럼
