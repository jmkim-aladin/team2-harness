# 스프린트 마감 프로세스 가이드

> YouTrack KB: DEV2-A-1097 | 상위: DEV2-A-1045 (운영 방안)

## 개요

스프린트 마감일(해당 월 마지막 영업일) 기준 **워킹데이 D-5**부터 시작하는 마감 프로세스입니다.
스프린트 데이터의 정합성을 확보하고, 다음 달 계획 수립의 기반을 만드는 것이 목적입니다.

- **시작 시점:** 스프린트 마감일 기준 워킹데이 D-5
- **완료 기한:** 스프린트 마감일 (D-day)

## D-5 | 티켓 상태 점검 및 이월 판단

> **담당:** 팀원 개인

### 본인 티켓 현황 파악

- 이번 스프린트에 배정된 본인 담당 티켓 전체 목록을 확인
- 완료된 업무는 **종료(Done)** 상태로 전환하며, **결과물 링크를 첨부** (문서: 지식베이스 링크 / 소스: 커밋 내역)

> **주의:** 결과물 링크가 확인되지 않는 티켓은 산출물이 부재한 것으로 간주되어 업무 이행 여부를 인정하기 어렵습니다. OKR 달성률 산정 시 미반영 사유가 될 수 있으므로, 반드시 검증 가능한 결과물 링크를 첨부합니다.

- 진행 중(In Progress) 업무는 마감일 내 완료 가능 여부를 판단
  - **완료 가능:** D-day까지의 완료 계획을 수립
  - **완료 불가:** 이월 대상으로 분류하고, 팀장에게 승인 요청

### 이월 대상 처리

- 팀장 사전 승인을 받은 후 티켓 상태를 **Backlog**으로 전환
- 이월 코멘트에 아래 항목을 필수로 기재
  - 이월 사유
  - 결정 내용 (당월 제외 확인)
  - 재확인 조건
  - 다음 확인 시점

> 상세 절차는 [업무 계획 변경 절차](./plan-change-process.md) 참조

## D-4 | 데이터 정합성 보완

> **담당:** 팀원 개인

### 스토리 포인트

- 본인 담당 최하위 노드 티켓에 SP가 모두 입력되어 있는지 확인
- SP가 미입력된 티켓은 Velocity 집계에서 누락되므로, 빠짐없이 입력

### 5W1H

- 이번 달 생성한 Feature 티켓에 5W1H 항목이 모두 기재되어 있는지 확인

### OKR 연결

- 계획 업무 티켓에 OKR/KR 연결(태그 또는 링크)이 표기되어 있는지 확인

## D-3 | 데이터 검증 및 Velocity 산출

> **담당:** 팀장 (검증) / 팀원 (귀책)

### 데이터 검증

팀장은 아래 항목을 스캔하여 **누락 현황을 팀 채널에 공유**합니다.
D-4까지의 데이터 정합성 작업은 팀원 본인의 책임이며, D-3 검증 시점에 누락이 발견된 건은 **해당 담당자의 미이행**으로 간주됩니다.

- **미종료 티켓 스캔**
- **SP 미입력 티켓 스캔**
- **5W1H 누락 점검**

> **운영 원칙:** D-3 검증에서 반복적으로 누락이 발견되는 경우, 해당 담당자의 스프린트 회고 시 개선 항목으로 다룹니다. 데이터 정합성의 1차 책임은 팀원 본인에게 있으며, 팀장은 검증과 피드백 역할을 수행합니다.

### Velocity 산출

- 팀 Velocity를 산출 (Done 처리된 최하위 티켓 SP 합산)
- 담당자별 Velocity를 산출
- 전월 대비 편차를 확인
  - ±20% 이내: 정상 범위
  - ±20% 초과: 원인을 파악하고 D-1 회의에서 논의

> 상세 산출 규칙: [Velocity 측정 및 활용 가이드](./velocity-guide.md)

## D-3 ~ D-2 | 다음 달 계획 티켓 작성

> **담당:** 팀원 개인

- 다음 달 진행할 Feature/Task 티켓을 생성
- 최하위 노드 티켓에 SP 산정을 완료
- 모든 티켓에 예상 시작 일자를 입력
- **3점(M) 이상 티켓은 산정 근거를 본문에 기재**
- **8점(XL) 이상 티켓은 하위 작업으로 분할**

## D-1 | 티켓 회의 — 다음 달 계획업무 확정

> **담당:** 팀 전체

### 수용량 검증 (팀장)

- 수용량 산정: 전월 Velocity × 0.8 × 가용률 보정 (휴가/공휴일/인원 변동)
- 다음 달 총 계획 SP가 산정 수용량을 초과하지 않는지 확인

### 백로그 우선순위 확정

- OKR 직결 티켓이 전체 계획 SP의 50% 이상인지 확인 (운영 상황에 따라 팀장 협의하에 유동 적용)
- 8점(XL) 이상 티켓의 분할 여부를 최종 확인
- 이월 백로그 중 재확인 시점이 도래한 건을 재논의

### 합의

- 다음 달 백로그를 최종 확정
- 담당자별 SP 배분 밸런스를 확인

## D-day | 스프린트 마감

> **담당:** 팀장 / 팀원

### 팀장

- 미종료 티켓을 최종 스캔하여 잔존 건의 이월 처리 완료
- 스프린트 데이터 확정
- Sprint Completion Rate, Velocity 편차, OKR 직결 완료율을 산출
- KR별 달성률 현황과 다음 달 잔여 목표량을 정리

### 팀원

- 확정된 본인 담당 백로그 티켓을 Open 상태로 전환하여 다음 달 스프린트 시작
- 월말까지 본인 OKR 문서에 KR별 진척률 코멘트를 게시

## 요약 타임라인

| 시점 | 담당 | 핵심 작업 |
|------|------|----------|
| **D-5** | 팀원 | 티켓 상태 정리, 이월 판단 및 팀장 승인 요청 |
| **D-4** | 팀원 | SP / 5W1H / OKR 연결 누락 보완 |
| **D-3** | 팀장(검증) / 팀원(귀책) | 데이터 검증 (미종료·미입력 스캔), Velocity 산출 |
| **D-3 ~ D-2** | 팀원 | 다음 달 계획 티켓 작성 및 SP 산정 |
| **D-1** | 팀 전체 | 티켓 회의: 수용량 검증, 백로그 확정 |
| **D-day** | 팀장 | 스프린트 마감 확정, 다음 달 스프린트 오픈 |
