## 김정민 개인 회고 연결점

### 1분기 자기평가 (60점 만점 중 ~14점)

| KR | 달성 | 핵심 이슈 |
|----|------|-----------|
| KR1 정량제 출시 | 0% (개발 완료, 운영 미배포) | 외부 의존(개발4팀 앱/웹뷰어) → **지표 설계 시 외부 의존 반영 부족** |
| KR2 ADMIN MSA | 40% (구축+문서 완료, 운영 미배포) | 핵심 기능 미착수 |

| KR4 코드 품질 | 80% (SP Git 100%, AI리뷰 완료) | PR 경유율 0% → 프로세스 도입 지연 |
| KR5 보안 DPA | 0% | 우선순위 밀림 |

### 보고서 인사이트와의 교차점

- **P1(지표 신뢰도)**: KR1은 개발 완료했으나 "운영 배포"가 지표 → 외부 의존 변수를 반영 못한 전형적 사례
- **T1(지표 재설계)**: 2분기 KR에는 "Win 인스턴스 100% 셧다운", "가용률 99.5%", "장애 0건" 등 **자체 통제 가능한 수치**로 개선됨
- **K1(연결 구조)**: OKR 외 성과(투비 AWS 이관, AASM)가 KR에 미반영 → 2분기에는 돌발 업무도 OKR에 매핑하는 구조 필요?

## 회고 진행 제안

### 1단계: 데이터 공유 (5분)

- 보고서 핵심 수치 공유 (28.1% → 53.6%, +25.5%p)
- "수치의 의미는 절대 달성률이 아닌 실행 가속도"임을 먼저 합의

### 2단계: 개인별 KPT 공유 (인당 5분)

각자 아래 질문에 답변 준비:
1. **Keep**: 3월에 가장 잘 된 업무 연결(OKR→산출물) 사례 1개
2. **Problem**: 본인 KR 중 "지표가 잘못 설정됐다"고 느낀 항목과 이유
3. **Try**: 2분기 KR에서 이미 개선한 점, 아직 부족한 점

### 3단계: 팀 논의 (15분)

- T1~T4 개선안 중 **4월에 바로 실행할 항목** 합의
- Velocity 측정 방법 및 추적 주기 결정
- 4월 말 KR 재조정 회의 일정 확정
