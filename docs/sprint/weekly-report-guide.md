# 주간업무 보고 가이드

> 출처: [YouTrack KB DEV2-A-692](https://aladincommunication.youtrack.cloud/articles/DEV2-A-692/7KO86rCE7JeF66y0)

## 1. 계획 업무 작성 원칙

기본적으로는 계획된 업무(팀 구성원 작성 티켓)를 기준으로 작성합니다.

다만, 이슈 규모가 크거나 팀 차원의 공유가 필요한 운영성 업무(사업부 작성 티켓)도 예외적으로 주간업무 보고에 포함할 수 있습니다.

---

## 2. 상태별 구분 기준

### ① 백로그

**정의**

- 현재 진행 중인 스프린트 종료 후 시작 예정인 업무
- 우선순위 변경으로 현재 스프린트에서 제외된 업무

**작성 티켓 기준**

- 티켓 유형: Epic, Feature
- 티켓 상태: Backlog
- 스프린트: 없음

**필수 작성 요소**

- 제목
- 담당자
- 5W1H 포함 티켓 링크

**세부 항목 작성**: 불필요

### ② 미진행 항목

**정의**

- 현재 진행 중인 스프린트에 포함되었으나 아직 시작하지 않은 업무
- 우선순위 조정으로 재정렬된 업무

**작성 기준**

- 티켓 유형: Epic, Feature
- 티켓 상태: Open
- 스프린트: 현재 진행 중인 스프린트

**필수 작성 요소**

- 제목
- 담당자
- 시작 예상 일자(~)
- 5W1H 포함 티켓 링크

**세부 항목 작성**: 선택

### ③ 진행 중 항목

**정의**

- 현재 스프린트에 포함되어 실제 작업이 진행 중인 업무
- 이전 스프린트에서 이월되어 계속 진행 중인 업무

**작성 기준**

- 티켓 유형: Epic, Feature
- 티켓 상태: In Progress
- 스프린트: 현재 진행 중인 스프린트(이월된 경우 과거 스프린트 이력 포함)

**필수 작성 요소**

- 제목
- 담당자
- (~)완료 목표 일자
- 5W1H 포함 티켓 링크

**세부 항목 작성**: 필수

### ④ 완료된 항목

**정의**

- 현재 진행 중인 스프린트에 포함되어 완료된 업무
- 이전 스프린트에서 완료되었으나, 지난 보고 이후 완료 처리된 업무

**작성 기준**

- 티켓 유형: Epic, Feature
- 티켓 상태: Fixed (또는 완료 상태)
- 스프린트: 해당 완료 시점의 스프린트 이력 유지

**필수 작성 요소**

- 제목
- 담당자
- 완료 일자
- 5W1H 포함 티켓 링크

**세부 항목 작성**: 선택

---

## 3. 세부(Task) 항목 관리 기준

Epic/Feature 하위 Task는 진행 상태를 명확히 구분하기 위해 말머리를 추가합니다.

**말머리 구분**

- (완료)
- (진행 중)
- (예정)

**작성 기준**

- 티켓 유형: Task
- 티켓 상태:
  - (완료) → Fixed
  - (진행 중) → In Progress
  - (예정) → Open
- 스프린트: 현재 진행 중인 스프린트

## 4. 템플릿

### 1) 백로그 항목

- **소제목** (담당자, 티켓 링크)

### 2) 미진행 항목

- **소제목** (시작 예상 일정, 담당자, 티켓 링크)
- **소제목** (시작 예상 일정, 담당자, 티켓 링크)

### 3) 진행 중 항목

- **소제목** (~목표 일정, 담당자, 티켓 링크)
  - (완료) 내용 (완료 일자, 담당자, 티켓 링크)
  - (진행 중) 내용 (~목표 일자, 담당자, 티켓 링크)
  - (예정) 내용 (시작 예정 일자~, 담당자, 티켓 링크)
  - 부가 내용
- **소제목** (~~~목표 일정~~ ~수정 목표 일정, 담당자, 티켓 링크) - 지연
  - 내용
  - 지연 사유

### 4) 완료된 항목

- **소제목** (완료 일자, 담당자, 티켓 링크)
- **소제목** (완료 일자, 담당자, 티켓 링크) - 지연
  - 지연 사유

### 5) 이슈사항

- **소제목** (티켓 링크(option))
  - 내용
- **소제목**
  - 내용

### 6) 기타

- **소제목**
  - 내용
- **소제목**
  - 내용
